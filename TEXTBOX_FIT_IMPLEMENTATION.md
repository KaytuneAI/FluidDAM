# 文本框适配格子实现说明

## 实现概述

根据 ChatGPT 的解决方案，我们实现了文本框"等比缩放 + 不超出格子（含合并单元格）"以及"文本框默认用白色不透明底"的功能。

## 主要修改

### 1. DrawingML.js 新增功能

#### 新增 `clampRectToCells` 函数
- **位置**: `src/utils/DrawingML.js` 第 208-281 行
- **功能**: 将文本框矩形钳制到单元格边界内（包括合并单元格）
- **参数**:
  - `rect`: 原始矩形 {x, y, w, h}
  - `dims`: 尺寸信息 {colOffsets: [], rowOffsets: []}
  - `merges`: 合并单元格数组 [{top, left, bottom, right}]
  - `anchor`: 锚点对象，用于判断锚点类型
- **返回**: 钳制后的矩形 {x, y, w, h}

#### 修改 `parseFillStyle` 函数
- **位置**: `src/utils/DrawingML.js` 第 45-91 行
- **新增参数**: `spNode` - 形状节点，用于检测是否为文本框
- **功能**: 为文本框添加默认白色不透明背景
- **逻辑**:
  - 检测到 `xdr:txBody` 节点时，判断为文本框
  - 文本框默认使用白色不透明背景 `{ fill: 'solid', color: '#FFFFFF', opacity: 1 }`
  - 即使 Excel 中明确设置为无填充，文本框仍然显示白色背景

#### 修改 `parseDrawingML` 函数

  - 使用钳制后的矩形创建文本框
  - 添加调试日志显示钳制前后的尺寸变化

## 实现原理

### 矩形钳制逻辑
1. **twoCellAnchor**: 矩形本身就覆盖了 from→to 的多格区域，直接使用
2. **oneCellAnchor**: 
   - 找到 from 所在的单元格（或合并单元格）
   - 计算单元格的像素边界
   - 将文本框矩形钳制到单元格边界内

### 文本框背景处理
1. **检测文本框**: 通过 `xdr:txBody` 节点判断是否为文本框
2. **默认白底**: 文本框默认使用白色不透明背景（opacity: 1）
3. **完全遮挡**: 白色背景像白纸一样完全遮挡住后面的图案，背景图案看不见
4. **保持原有样式**: 如果 Excel 中已有填充样式，则保持原有样式

## 使用效果

### 文本框适配格子
- 文本框会自动适配到单元格边界内
- 支持合并单元格的边界约束
- 文本会在文本框内自动换行，不会溢出

### 文本框背景
- 文本框默认显示白色不透明背景（opacity: 1）
- 像白纸一样完全遮挡住后面的图案，背景图案看不见
- 与 Excel 中的视觉效果保持一致
- 支持自定义背景色和透明度
