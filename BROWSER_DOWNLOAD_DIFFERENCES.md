# 浏览器文件下载差异说明

## 问题描述
在Edge浏览器中点击"保存画布"时，文件直接保存而不弹出保存对话框，而在Chrome中会弹出保存对话框。

## 原因分析

### 1. 浏览器安全策略差异
- **Chrome**: 更严格的安全策略，要求用户明确确认下载操作
- **Edge**: 相对宽松的策略，允许程序化下载直接执行

### 2. 事件处理机制不同
- **Chrome**: 需要用户交互事件（如真实的点击）才能触发下载
- **Edge**: 对程序化创建的点击事件更宽容

### 3. 下载API实现差异
- **Chrome**: 使用标准的HTML5 download属性
- **Edge**: 可能使用不同的内部实现机制

## 解决方案

### 1. 浏览器检测
```javascript
const isEdge = navigator.userAgent.includes('Edg/');
const isChrome = navigator.userAgent.includes('Chrome/') && !navigator.userAgent.includes('Edg/');
```

### 2. 差异化处理
```javascript
if (isEdge) {
  // Edge专用方法
  const event = new MouseEvent('click', {
    view: window,
    bubbles: true,
    cancelable: true
  });
  a.dispatchEvent(event);
} else {
  // Chrome和其他浏览器
  a.click();
}
```

### 3. 统一工具函数
创建了 `downloadUtils.js` 来统一处理不同浏览器的下载逻辑。

## 注意事项

1. **用户设置**: 不同用户的浏览器下载设置可能影响行为
2. **安全策略**: 某些企业环境可能有更严格的安全策略

4. **文件名**: 避免使用特殊字符，确保跨平台兼容
