# 下载行为更新说明

## 问题描述
之前的版本在Chrome中会同时显示保存对话框和绿色通知条，造成重复提示。

## 解决方案
根据浏览器类型采用不同的通知策略：

### 浏览器行为差异

#### 🟢 Edge / Firefox
- **行为**: 直接下载到默认文件夹
- **通知**: 显示绿色成功通知条
- **原因**: 这些浏览器不会弹出保存对话框，用户需要反馈

#### 🔵 Chrome / Safari  
- **行为**: 弹出保存对话框
- **通知**: 不显示额外通知
- **原因**: 浏览器本身已提供保存反馈，避免重复提示

#### 🟡 其他浏览器
- **行为**: 根据浏览器默认行为
- **通知**: 显示绿色成功通知条
- **原因**: 保守策略，确保用户得到反馈

## 技术实现

### 浏览器检测
```javascript
const browser = detectBrowser();
// 返回: 'edge', 'chrome', 'firefox', 'safari', 'ie', 'unknown'
```

### 差异化处理
```javascript
if (browser === 'edge' || browser === 'firefox') {
  // 直接下载，显示通知
  setTimeout(handleSuccess, 200);
} else if (browser === 'chrome' || browser === 'safari') {
  // 弹出对话框，不显示通知
  a.click();
} else {
  // 其他浏览器，显示通知
  setTimeout(handleSuccess, 200);
}
```

## 用户体验

### Edge用户
1. 点击"保存画布"按钮
2. 文件直接保存到下载文件夹
3. 看到绿色通知条确认保存成功

### Chrome用户
1. 点击"保存画布"按钮
2. 浏览器弹出保存对话框
3. 用户选择保存位置并确认
4. 无额外通知（避免重复提示）

### Firefox用户
1. 点击"保存画布"按钮
2. 文件直接保存到下载文件夹
3. 看到绿色通知条确认保存成功

## 优势

### ✅ 避免重复提示
- Chrome用户不会看到重复的保存确认
- 每个浏览器都有最适合的反馈方式

### ✅ 更好的用户体验
- Edge/Firefox用户得到明确的成功反馈
- Chrome用户通过原生对话框获得反馈

### ✅ 跨浏览器一致性
- 所有浏览器都能正常工作
- 根据浏览器特性优化体验
