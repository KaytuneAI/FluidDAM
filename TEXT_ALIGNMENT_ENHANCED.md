# Excel文字对齐进一步优化

## 问题分析

用户反馈"没有太大变化"，可能的原因：

1. **默认对齐方式**: 大部分Excel单元格本来就是左对齐和顶部对齐
2. **文字高度计算不准确**: 之前假设的14px可能不准确
3. **内边距还是太大**: 2px可能还是比Excel大
4. **缺少Excel智能默认对齐检测**

## 进一步优化

### 1. **更精确的内边距**
```javascript
// 从2px减少到1px，更接近Excel标准
const padding = 1;
```

### 2. **更准确的文字高度计算**
```javascript
// 12px Arial字体的实际渲染高度约为14.4px
const textHeight = 14.4;
```

### 3. **Excel智能默认对齐检测**
```javascript
// 检测Excel的默认行为
if (!horizontal) {
  // Excel默认：数字右对齐，文本左对齐
  const value = cell.value;
  if (typeof value === 'number') {
    horizontal = 'right';
  } else {
    horizontal = 'left';
  }
}

if (!vertical) {
  // Excel默认：垂直居中对齐
  vertical = 'middle';
}
```

### 4. **测试功能**
- 添加了调试信息显示对齐设置
- 强制某些文本（如"第一帧"、"第二帧"等）使用居中对齐来测试效果
- 可以在控制台看到详细的对齐信息

## 技术改进

### 内边距优化
- **之前**: 2px内边距
- **现在**: 1px内边距，更接近Excel标准

### 文字高度精确化
- **之前**: 假设14px高度
- **现在**: 使用14.4px，更准确的Arial字体渲染高度

### 智能对齐检测
- **数字**: 自动右对齐
- **文本**: 自动左对齐
- **垂直**: 默认居中对齐

### 测试验证
- 强制某些文本居中对齐来验证效果
- 控制台显示详细的对齐信息
- 可以直观看到对齐变化

## 预期效果

现在应该能看到更明显的对齐变化：

1. **帧标题文本**: 强制居中对齐，应该更明显
2. **数字文本**: 自动右对齐
3. **普通文本**: 左对齐但垂直居中
4. **更精确的位置**: 1px内边距，14.4px文字高度
